⚡ DocChat — How It Works

Two flows: Upload PDF → Ask Question

📤 Flow 1 — PDF Upload
1
📄
User uploads PDF
Frontend sends file to /api/documents via multipart form.
POST /api/documents
Content-Type: multipart/form-data
file: resume.pdf
▼
2
🔍
Text Extraction
Backend uses pdf-parse library to extract all raw text from the PDF.
"Gradient descent is an optimization algorithm used to minimize a loss function by iteratively..."
▼
3
✂️
Chunking
Text is split into overlapping pieces (~1500 chars each, 300 char overlap) so no context is lost.
chunk_1
chunk_2
chunk_3
chunk_4
chunk_5
· · ·
chunk_42
▼
4
🧮
Embeddings Generate
Har chunk ko numbers ke array (vector) mein convert kiya jata hai using Gemini embedding model.
chunk_1 → [0.12, 0.87, 0.34, ...]
chunk_2 → [0.55, 0.21, 0.90, ...]
384 numbers per chunk
▼
5
🗄️
Save to MongoDB
Chunks + embeddings MongoDB mein save ho jaate hain. Status "ready" ho jaata hai.
🍃 MongoDB Atlas · 42 chunks saved
vs
💬 Flow 2 — Ask Question
1
❓
User types question
Frontend sends question to /api/chat with the document ID.
POST /api/chat
{
documentId: "abc123",
question: "What is gradient descent?"
}
▼
2
🧮
Question → Embedding
Question ko bhi usi model se vector mein convert kiya jata hai.
"What is gradient descent?"
→ [0.44, 0.91, 0.13, ...]
▼
3
🎯
Find Similar Chunks
Question vector aur har chunk vector ke beech cosine similarity calculate hoti hai. Top 5 chunks select hote hain.
chunk_7 ✦ 0.91
chunk_9 ✦ 0.87
chunk_11 ✦ 0.74
chunk_3 0.12
chunk_1 0.08
▼
4
🤖
Send to Groq (LLM)
Top chunks + question Groq ke LLM (Llama 3) ko bheje jaate hain as context.
System: Answer ONLY from these excerpts:
[1] chunk_7 text...
[2] chunk_9 text...
User: What is gradient descent?
▼
5
✅
Answer + Citations
LLM grounded answer deta hai with [1][2] citation numbers. Frontend mein green cards mein source dikhta hai.
"Gradient descent minimizes loss
by moving opposite to gradient [1].
Learning rate controls step size [2]."
Upload Flow (one time per document)
Query Flow (every question)
Gemini = Embeddings
Groq = LLM Answer
